# macau6hbd-pages

本仓库用于归档和发布多个独立的 HTML 页面，作为静态页面的托管与分发空间。仓库本身不针对任何单个域名或具体网站，仅提供页面文件的存储与访问能力。

## 项目简介

macau6hbd-pages 是一个通用的 HTML 页面归档仓库，旨在集中管理一批独立的静态页面资源。这些页面可能用于不同的场景或用途，仓库仅作为它们的技术载体，不附加任何特定主题或推广内容。

## 目录说明

```
.
├── index.html          # 仓库根目录入口页面
├── pages/              # 存放归档的 HTML 页面文件
│   └── example.html    # 示例页面
└── assets/             # 页面引用的静态资源（CSS、JS、图片等）
    └── style.css
```

- **`pages/`**：核心目录，存放所有归档的 HTML 页面文件，每个文件对应一个独立页面。
- **`assets/`**：存放页面所需的公共资源文件，如样式表、脚本、图片等。
- **`index.html`**：仓库根路径下的默认页面，可作为导航或说明页。

## 页面归档说明

- 所有归档页面均为独立的 HTML 文件，彼此之间无直接依赖关系。
- 页面内容由仓库维护者自行定义，本仓库仅负责提供文件托管和版本管理。
- 页面文件命名建议使用小写英文字母、数字和连字符，避免特殊字符。
- 每个页面应确保引用资源路径正确，推荐使用相对路径引用 `assets/` 目录下的资源。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 将静态资源放入 `assets/` 目录，并在页面中通过相对路径引用。
4. 提交更改并推送至 GitHub，仓库将自动发布页面（若已开启 GitHub Pages）。

> 注意：若需通过 GitHub Pages 访问页面，请确保仓库设置中已正确配置发布源（通常为 `main` 分支或 `docs/` 目录）。

## 维护说明

- 本仓库由维护者不定期更新，添加或清理页面内容。
- 欢迎通过 Issues 提交关于页面结构或资源管理的建议。
- 不接收涉及具体域名、商业推广或违规内容的页面提交。
- 维护者保留对仓库内容的最终管理权，包括但不限于删除或修改页面。
